## Description of calibratable parameters

A detailed description of the calibratable parameters is given below. However, not all parameters should probably be included in calibration because this would lead to a large calibratabale space. 

It is recommended to calibrate:

- `theta_e` for the top layer  
- `alpha` for the top two layers  
- `Ks` for the top two layers  
- `n` for the top two layers  
- `field_capacity_psi`  
- `a`  
- `b`  
- `frac_to_GW`  
- `spf_factor`

Note that it is not necessarily recommended to calibrate maximum ponded head or theta_r.

Note that theta_r must be less than than theta_e. This is set per soil layer, in the .dat file in the data directory.|
| theta_e | - | 0 < theta_e < 1, <br> theta_r < theta_e | 0.3 < theta_e < 0.8 | Soil layer | theta_e is the maximum volumetric water content that a soil layer can naturally attain. Note that theta_e must be greater than theta_r. This is set per soil layer, in the .dat file in the data directory.|

Parameters that are specified per soil layer can either be scalar values with double precision (in the event the model is run with 1 layer) or vectors of doubles (in the event that the model is run with more than 1 layer), whereas parameters that are specified for the entire model domain are scalars with double precision. 

Stability testing efforts have included varying initial conditions, where the initial condition throughout the soil moisture profile is set with a single value for capillary head, which must be 0 or greater (again note that these are absolute values of negative numbers and indicate unsaturated soils). This value can be set in the config file. 

We recommend that practical parameter values for calibration efforts of real soils use ranges that are for some parameters somewhat more restricted than the ones used in stability testing. For example, while it was desirable to test theta_e values up to 0.8, we do not expect to often see theta_e values of this magnitude in nature. Further, randomly sampling parameter values within even restricted ranges to build parameter sets could theoretically yield unrealistic soils (for example, a parameter set could have a K_s value of 0.01 cm/h, indicative of clay, and an n value of 2.5, indicative of sand).

Finally, please note that LGAR itself does not simulate direct fluxes from the vadose zone to groundwater (recharge) in a physically meaningful way. LGAR technically simulates recharge because sometimes a fast moving wetting front will partially go below the lower boundary of the LGAR domain. When this happens, the fraction of the wetting front that does this gets removed from the model domain and is said to be lost to deep groundwater. In arid or semi arid environments (with a sufficiently deep simulated vadose zone or low enough conductivity soils), this will only rarely happen, and when it does, the wetting front crossing the lower boundary will be moving slowly and therefore simulated fluxes through the lower boundary will be small. If your recharge fluxes at the end of the simulation are substantial, it could be the case that you should increase your vadose zone thickness, use a smaller (possibly fixed) time step, or use less conductive soils. If your precipitation is substantially greater than your PET for the simulation period, it is likely that you are not running the model in an arid or semi arid area. In this case, if you are interested in physically meaningful groundwater fluxes, then the LGARTO branch might be of interest, where both a shallow water table lower boundary condition (in TO mode) and a free drainage lower boundary condition (not in TO mode) are available, although the LGARTO branch has not been fully developed for use as a catchment scale model and is more appropriate to be used as a soil column model.
